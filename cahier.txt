1. PRESENTATION GENERALE
1.1 Contexte
Dans une ere de forte numerisation du commerce, de nombreux artisans et createurs marocains peinent a se faire une place en ligne en raison des barrieres techniques et des couts lies a la creation d'un site web. Le projet SOUK a ete imagine pour pallier ce probleme. SOUK est une plateforme SaaS (Software as a Service) Multi-tenant (comme Shopify ou YouCan) qui permet a n'importe quel commercant de creer, gerer et propulser sa propre boutique e-commerce instantanement.
1.2 Objet du document
Ce document constitue le cahier des charges officiel regissant la conception technique et fonctionnelle du projet SOUK. Il sert de contrat de developpement definissant les specifications a respecter, les technologies a utiliser ainsi que l'horizon de livraison du Minimum Viable Product (MVP).
1.3 Equipe projet
Developpeur(s) principal(aux)
: Conception UI, Integration React, et Developpement de l'API Laravel.
Encadrant(e) pedagogique
: Professeure Amira (Validation et suivi academique).
1.4 Stack technique
Afin d'assurer une experience luxueuse, rapide et evolutive, le projet repose sur des technologies de pointe :
Frontend
: React.js via Vite, React Router DOM, sans librairie CSS lourde (le styling est fait en natif JS Vanilla CSS).
Backend
: API REST sous le framework Laravel 11.
Base de donnees
: Relationnelle MySQL de haute disponibilite.
Authentification
: Gestion par Tokens JWT via Laravel Sanctum.
2. OBJECTIFS DU PROJET
Creer une identite Premium
: Contrairement aux plateformes e-commerce generiques, SOUK doit degager une identite visuelle "Maroc de luxe" (couleurs Gold, Copper, Zellige subtil, typographies soignees).
Zero barriere technologique
: Permettre l'onboarding d'un vendeur (Creation de boutique, personnalisation, premier produit publie) en moins de 2 minutes.
Architecture SaaS robuste
: L'URL de chaque vendeur (/store/{slug}) doit isoler ses propres donnees et produits pour eviter de melanger son panier ou son catalogue avec le reste des marchands SOUK.

4. SPECIFICATIONS TECHNIQUES
4.1 Architecture
Architecture en micro-services logiques : une API Backend (Headless) pure et une Application Page Unique (SPA) sous Vite/React pour la partie cliente. La gestion Multi-tenant s'appuie sur le parametre dynamique du "slug" dans les URLs API (/api/{slug}/products).
4.2 Base de donnees
Structuration
: Relationnelle en base de donnees MySQL.
Tables maitresses
: Users (Vendeurs), Stores, Products, Orders, et un repertoire Clients (Acheteurs globaux).
Isolement des donnees
: La relation store_id devient la cle etrangere omni-presente pour garantir l'isolement "Tenants".
4.3 Securite
Authentification Dual-JWT
: Utilisation de tokens isoles (l'un d'eux souk_token pour le vendeur, et souk_client_token pour l'acheteur global).
Protection CSRF/CORS
: Le Backend autorise uniquement le Frontend SOUK.

7. EXCLUSIONS (NON INCLUS)
Le Minimum Viable Product presente lors du projet de synthese n'incorpore pas :
Paiement reel
: La connexion reelles en Production avec l'architecture CMI pour traiter les vraies cartes bancaires. Le "Paiement en Ligne" fonctionnera en simulation.
Applications mobiles
: Le developpement natif Flutter / React Native d'applications mobiles (Le site PWA etant cependant totalement fluide sur telephone).
